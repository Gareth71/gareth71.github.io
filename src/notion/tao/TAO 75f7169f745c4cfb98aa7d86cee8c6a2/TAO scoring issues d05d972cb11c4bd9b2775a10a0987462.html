How RESPONSEs work

When we write questions the input in the question is linked to the RESPONSE by using the RESPONSE identifier. The number in the identifier is only there to match the input to the answer and has no relevance to the order.

<responseDeclaration identifier="RESPONSE_5" cardinality="single" baseType="string">
<correctResponse>
<value><![CDATA[I]]></value>
</correctResponse>
</responseDeclaration>

Example question. This one will link to RESPONSE_5

<p><strong>1  </strong> Astronauts brought rocks back from the moon  <textEntryInteraction responseIdentifier="RESPONSE_5" base="10" placeholderText=""/> </p>

There are often more RESPONSEs than questions and I am guessing this is due to the content of the activity being edited. I think when you delete an input its RESPONSE is not removed but it loses its content.

<responseDeclaration identifier="RESPONSE_1" cardinality="single" baseType="identifier"/>

Copying and pasting the activities content probably also results in the order being messed up or maybe the order that you add the responses to the questions sets their order.

For some reason audio is also tagged with a response, the below is an audio player. This doesn’t seem to have any effect on the scoring.

<mediaInteraction responseIdentifier="RESPONSE_5" autostart="false" minPlays="0" maxPlays="0" loop="false" class="pause">
<object data="assets/b8600c5d631218176e644.mp3" type="audio/mpeg" width="984" height="30"/>
</mediaInteraction>
