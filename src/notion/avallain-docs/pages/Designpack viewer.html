Installing the Previewer

Add VIEWER_GR to your machine.

In terminal cd to VIEWER_GR

Avallains documentation says you need node version 16.1.0 but I am not sure if it applies to the viewer or just the compiler, best to stick to 16.1.0

nvm use 16.1.0

Install the server, Avallain recommends to do this with brew brew install http-server or globally with npm npm install http-server -g

To get certain activities to work like voice recording you will need certificates which requires installing openssl We might be able to omit this step if we don’t care about recording but will need to alter the package.json script

Add openssl with brew brew install openssl@1.1
With terminal pointing to VIEWER_GR run this command

openssl req -newkey rsa:2048 -new -nodes -x509 -days 3650 -keyout key.pem -out cert.pem

Fill in the details that it asks for until you get to Common name then add 127.0.0.1

Link to compiled design packs

In terminal cd to the dp folder

Run these commands to create sym links to where your design packs are compiled. If you have created them in a different location you will need to alter the path.

ln -s ~/code/avallain/author/designpacks/root root
ln -s ~/code/avallain/author/designpacks/root/tmp/compileddp root-compiled
ln -s ~/code/avallain/author/designpacks/child1/tmp/compileddp child1
ln -s ~/code/avallain/author/designpacks/child2/tmp/compileddp child2
ln -s ~/code/avallain/author/designpacks/child3/tmp/compileddp child3

The viewer doesn’t compile the design packs so you will have to do that separately.

Creating your LO javascript file

The data for an LO is stored in a javascript file like this. Make a new one for each LO and populate it with data from previewing it in Author.

export default {
  // LOOptions content copied from the LO document head
  options: // window.LOInfo.LOOptions
,
	// replace below with data.do Response
	// replace "=" with ":"  so you have ajaxData: {
	ajaxData: {
	}
}

  1. With the Network tab of developer tools open preview the LO and locate the data.do file.

    Select everything in the Response tab and copy, paste into the lo.js file.
    Open Network tab and preview LO then search for data.do in the search field. Select data.do and then copy the contents of the Response tab.

  1. In the console type this window.LOInfo.LOOptions copy results and paste after options:

    Add an inch mark at start and end of the pasted string

Store the file in the lo folder.

We are now ready to view some LOs

In terminal pointing to VIEWER_GR run npm run start

Open localhost:8080/?package=child2&lo=lo1.js

https://127.0.0.1:8080/?package=child2&lo=lo1.js in your browser where child2 is the name of the compiled design pack you want to use and lo1.js is the name of the filewith your LO data in it.

Chrome will probably refuse to preview so you ca try adding this in a new tab chrome://flags/#allow-insecure-localhost but it didn’t work for me so I also had to go back to the localhost tab and type “thisisunsafe”. You don’t have to put your cursor anywhere, as long as the tab is the active one it detects the key presses.

All being well you should see your LO.
