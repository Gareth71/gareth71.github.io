Code base

  1. Author: Online editing tool, that creates LOs
  1. Engine: buckminster Also called player, and is responsible for making learning objects work in the browser.
  1. Compiler ambros Converts design-packs into a format the engine can use.
  1. Design Packs: Avallain Base HTML/CSS/Javascript/images/fonts that gives an LO the desired look and feel

Code releases

Avallain have a 12 week cycle

  • 2 releases on Stage where TCE do testing
  • 1 on Stage where de test design packs
  • Finally onto Production

Structure

Config.xml

Configuration properties of the DP in Java properties XML format.

  • design pack parent / parent git url / parent tag
  • editor config
  • define layouts
  • Create Class names that can be applied later when authoring.

head.html

These are not attached to the head as you might expect but bundled

  • CSS
  • JS

images

  • images
  • icons.svg (most icons are stored here in a svg sprite sheet)

js

  • All javascript files, mainly for activities that run on top of the Author 5 engine
  • config.js additional preferences can be set here.
  • fonts.js

less

Split up into multiple LESS files

locales

  • JSON files for different languages

resources

  • Storage for source items like PSD files (not good having it here)

How it works

  1. We create a branch in our repository.
  1. Set its parent
  1. Go to Avallain and create a design pack and link the repository
  1. Make changes to it
  1. Commit to GIT
  1. Apply a tag to them
  1. Push to Repro
  1. Go to Avallain PreProd as an Admin and update tag and compile.
  1. Check
  1. Go to Avallain Prod as an Admin and update tag and compile.

Compilation happens on the server in the admin section or can happen locally if you set up their local development environment.

On compilation it pulls in the parent design pack or parents as you can have grand parents great grand parents etc and merges them all together.

Most features overwrite parent ones eg icons.svg in our design pack over-rides the one in the base design pack.

All the less files from all the design packs are compiled together with child ones being added in later. This allows for variables to ripple back up from the child and for files to be turned off.

Cons

  • LESS rather than SCSS
  • No maps unless doing local compilation so tracking down errors is difficult.
  • Relies too heavily on mixins which create unessasary styles and make tracking down the source of code difficult.
  • SVG sprite sheet is not ideal. Better if we could insert the SVGs into the DOM so they can be manipulated with CSS. Hub has a nice system for inserting SVGs with “use”.
  • Class names are inconsistent and require quite lengthy selectors
  • Avallain Base still has a lot of CSS in it
    • I have seen a new version in which they abandoned creating lots of variables and made a striped back base design pack using BEM naming and only the minimum required LESS
