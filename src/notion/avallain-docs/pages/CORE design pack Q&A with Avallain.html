DP React Components Build Process

In summary React Build process will analyze the config.json, copy the React components that are not implemented (overwrite) on the actual DP from their ancestors, create a bundle library with all components, and clean the copied components from actual DP.

Configure DP to use React Activities

To configure a DP to use React Component, even when not overwritting then, the DP should have:

  • Build configuration Files (copy from Core)
    • - /package.json
    • - /package-lock.json
    • - /webpack.config.js
    • - /.babelrc
  • React Folder: (copy react folder and index.js)
    • - /react/
    • - /react/index.js
Do not change the /react/index.js, that file should always have all the react components overwritten or not. There you should have all the components and imports as it on root DP (Core)
  • Build Folder (copy from core)
    • /build/
    • /build/build-react.js
    • /build/utils.js
    • /build/config.json (to configure refer to config.json section below)

config.json

DP need to have a config.json with the components array where DP indicates the React Components that are implemented or overwritten in current DP (Core should have all components).

If the DP use react component but do not overwrite them (the JSX), just add an empty array to the components variable:

{    "components": []}

If DP overwrite some react component logic, just add the path of the overwritten components to config.json variable, for example:

{    "components": ["activities/ICCrossword", "CompC"]}

As said the script will copy from Core the React components that are not overwritten on the DP and create a bundle with the ones present on the config.json.

Build Process

Once the DP is correctly configure you can compile it using Ambros:

ambros compile

For a manual build follow the instructions below

  1. Install necessary Node Modules npm ci
  1. Configure config.json as explained.
  1. Run the build script, use the ancestors parameter as Array with the paths of DP ancestors (order is important, from the most distant ancestor to the closest):
    • npm run react:build for production
    • npm run react:build:dev for development
    • npm run react:build -- --ancestors ../root ../child-1 production with ancestors
    • npm run react:build:dev -- --ancestors ../root ../child-1 development with ancestors > Compiled react library will be located at /tmp/compileddp/react as the other compiled files.
Alternative you can use ambros compile, that will analyze the ancestors and compile React for the folder where is executed
